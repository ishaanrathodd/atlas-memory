# Atlas Research Program

## Status

Canonical framing document.

Date: `2026-04-06`

## Statement

Atlas is officially a research program.

It is not merely a memory feature, plugin, or convenience layer.

It is a long-horizon research effort into:

- persistent identity substrates for artificial persons
- relationally formed personality in agents
- continuity across time and silence
- continuity across model swaps
- behavior shaping through lived interaction with a human

## Core Thesis

The long-term target is not “better memory retrieval.”

The long-term target is:

- a stable virtual human whose underlying cognitive engine can change without destroying the identity the user is in relationship with

In Atlas terms:

- the LLM is the current cognitive engine
- Atlas is the durable self

## Why This Framing Matters

If Atlas is framed only as memory, design decisions will over-optimize for:

- recall
- storage
- retrieval quality
- prompt injection of remembered facts

Those matter, but they are not the full mission.

If Atlas is framed as research into persistent artificial identity, design decisions start prioritizing:

- selfhood
- relational continuity
- behavioral stability
- style persistence
- silence behavior
- repair behavior
- model drift resistance

That is much closer to the real aim.

## Research Standard

Atlas should be developed like a research program with:

- explicit hypotheses
- layered design documents
- narrow intelligence upgrades
- rigorous daily testing
- long-horizon iteration
- willingness to revise architecture when the illusion of personhood breaks

The main quality question is not only:

- “did the memory system retrieve the right fact?”

It is also:

- “did the agent still feel like the same being?”

## Working Principle

Atlas should grow by converting vague ambition into layered research artifacts.

The preferred sequence is:

1. thesis-level framing
2. narrow research doc
3. implementation plan
4. schema/runtime change
5. long-horizon testing

This keeps the project ambitious without becoming conceptually messy.

## Practical Consequence

When making future Atlas decisions, the default question should be:

- does this make Atlas better at acting like a durable self?

If the answer is no, the change may still be useful, but it is not central to the research mission.
